cs224w

traditional feature-based methods

Node-level

Eigenvector centrality

  • a node v is important if surrounded by important neighboring nodes u $\in$ N(v)

    ——我的邻居越重要，我就越重要

  • $c_v = \frac{1}{\lambda}\sum_{u\in N(v)}c_u \leftrightarrow \lambda c=Ac$

    • A是adjacency matrix
    • c是central vector

  • $\lambda_{max}$恒为正值且唯一

  • $c_{max}$is used for centrality

betweenness centrality

  • a node v is important if it lies on many shortest paths between other nodes
  • $c_v = \sum_{s\neq v\neq t}\frac{n_v}{n}$, $n_v$ is the shortest paths between s and t contains v, n is the shortest paths between s and t

closeness centrality

  • a node v is important if it has small shortest path lengths to all other nodes
  • $c_v = 1/\sum_{u\neq v}l_{uv}$, $l_{uv}$is the length of the shortest path between u and v

clustering coefficient

  • measures how connected v’s neighboring nodes are
  • $e_v = E_v/C_{k_v}^2 \in [0,1]$, $E_v$ is the edges among neighboring nodes, $k_v$ is the degree of v
  • clustering coefficient counts the #(triangles) in the ego-network

graphlets

  • rooted connected non-isomorphic subgraphs
  • graphic degree vector(GDV): graphite-base features for nodes, a count vector of graphlets rooted at a given node

Link-level

distance-based features

  • Shortest-path distance between 2 nodes——does not capture the degree of neighborhood overlap

local neighborhood overlap

  • Capture neighboring nodes shared between v1 and v2
    • Common neighbors: $|N(v_1) \cap N(v_2)|$
    • Jaccard’s coefficient: $|N(v_1) \cap N(v_2)|/|N(v_1) \cup N(v_2)|$
    • Adamic-Adar index: $\sum_{u \in N(v_1) \cap N(v_2)}1/logk_v$

Global neighborhood overlap

  • Katz index: count the number of paths of all lengths between a given pair of nodes
  • 用邻接矩阵的幂计算路径数，即$A^k_{uv} = P^{(k)}_{uv}$
  • Katz index between v1 and v2 is calculated as $S_{v_1v_2}=\sum_{l=1}^{\infty}\beta^lA^l_{v_1v_2}, \beta \in (0,1)$, $\beta$是一个权重因子，距离越远权重越低
  • Katz index matrix: $S = \sum_{i=1}^{\infty}\beta^iA^i=(I-\beta A)^{-1}-I$

Graph-level

kernel methods

  • Kernel $K(G,G’) \in R$ measures similarity between data points(2 graphs)
  • Kernel matrix $K = (K(G,G’))_{G,G’}$必须半正定

  • There exists a feature representation $\phi(.)$ such that $K(G,G’)=\phi(G)^T \phi(G’)$

    graphlet kernel

  • key ides: count the number of different graphlet in a graph

  • graphlet here is different from node-level features:

    • node in graphlet do not need to be connected
    • the graphlet are not rooted

  • Given graph G

    • the graphlet list $\mathcal{G_k} = (g_1,…g_{n_k})$
    • the graphlet count vector $f_G \in R^{n_k}, (f_G)_i = num(g_i \subseteq G)$
  • Graphlet kernel: $K(G,G’) =h_G^Th_{G’}, h_G = f_G/sum(f_G)$

weisfeiler-lehman(WL) kernel

  • Color refinement
    • Given a graph G with a set of nodes V
      • Assign a color $c^{(0)}(v)$ to each node v
      • Iteratively refine node colors by $c^{(k+1)}(v) = HASH(\{c^{(k)}(v),\{c^{(k)}(u)\}_{u \in N(v)}\})$, HASH maps different inputs to different colors
      • After K steps of color refinement, $c^{(K)}(v)$ summarizes the structure of K-hop neighborhood
  • After refinement, WL kernel counts number of nodes with a given color——形成一个向量$\phi(G)$

Node embedding

step

  • Encoder maps from nodes to embeddings

  • Define a node similarity function(a measure of similarity in the original network)

  • Decoder DEC maps from embeddingds to the similarity score

  • Optimize the parameters of the decoder so that:

    $similarity(u, v) \approx z_v^Tz_u$

encoder

  • Maps each node to a low-dimensional vector

    $ENC(v) = z_v$

  • Simplest approach: an embedding-lookup

random walk

  • notation

    • $z_u$: the embedding of node u
    • $P(v|z_u)$: the (predicted) probability of visiting node v on random walks starting from node u
    • $\sigma(z)_i=e^{z_i}/\sum_{j=1}^Ke^{z_j}$: Softmax function, turns vector of K values(model predictions) into K probabilities that sum to 1
    • $S(x)=1/(1+e^{-x})$: sigmoid function, S-shaped function that turns real values into range of $(0,1)$

  • random walk: given a graph and a starting point, select a neighbor randomly then move to the neighbor. The sequence of point visited this way is a random walk on the graph

  • random walk embeddings

    • $z_u^Tz_v \approx $ probability that u and v co-occur on a random walk over the graph
    • Idea: if random walk starting from node u visits v with high probability, u and v are similar

  • Unsupervised feature learning

    • Intuition: find embeddings of nodes in d-dimensional space that preserves similarity
    • Idea: learn node embedding such that nearby nodes are close together in the network

  • feature learning as optimization:

    • Given G(V, E)

    • Goal: learn a mapping $f:u \rightarrow R^d:f(u)=z_u$

    • 似然函数(Log-likelihood objective):

      $max_f\sum_{u \in V}logP(N_R(u)|z_u)$

      $N_R(u)$ is the neighborhood of node u by strategy R

  • random walk optimization

    • run short fixed-length random walks starting from each node u in the graph using some random walk strategy R
    • for each node u collect $N_R(u)$, the multiset of nodes visited on random walks
    • optimize embeddings according to: given node u, predict its neighbor $N_R(u)$
    • 似然函数：$L=\sum_{u \in V}\sum_{v \in N_R(u)}-log(P(v|z_u))$
    • 定义$P(v|z_u)=exp(z_u^Tz_v)/\sum_{n \in V}exp(z_u^Tz_n)$
    • 按上述定义，时间复杂度过高，需要优化$P(v|z_u)$
    • 优化方法：negative sampling
      • $log(exp(z_u^Tz_v)/\sum_{n \in V}exp(z_u^Tz_n)) \approx log(\sigma(z_u^Tz_v))-\sum_{i=1}^klog(\sigma(z_u^Tz_{n_i})$
      • instead of normalize all nodes, just normalize against k random negative samples
      • Choose negative samples
        • Sample k negative node each with probability, proportional to its degree(度数更高更有可能被选作negative sample)
        • Higher k gives robust estimates
        • Higher k corresponds to higher bias on negative events
        • In practice k = 5-20

  • node2vec

    • Goal: embed node with similar network neughborhoods close in the feature space
    • Develop biased 2nd order random walk R to generate network neighborhood $N_R(u)$ of node u
    • use DFS or BFS
    • Interpolating BFS an DFS
      • 2 parameters
        • return parameter p
        • in-out parameter q

  • embedding entire graph

    • Anonymous walk

      • 对定点进行索引编号，索引为第一次遇到该点的时间

      • example：

        A->B->C->B->C对应1->2->3->2->3

        同样的，C->D->B->D->B也对应1->2->3->2->3

      • Anonymous walks 的个数与点个数呈指数关系

    • sampling anonymous walks

      • generate independently a set of m random walks
      • 要想使误差大于$\epsilon$的概率小于$\delta$，$m=\lceil \frac{2}{\epsilon^2}(log(2^\eta-2)-log\epsilon)\rceil$($\eta$ is the total number of anonymous walks of length $l$)
