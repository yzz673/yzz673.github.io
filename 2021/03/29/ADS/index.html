    红黑树

    定理1

    • $N$个内点的红黑树最高$2log(N+1)$

      Proof

      令$sizeof(x)$表示根为x的树的内点个数，$bh(x)$表示根为x的树的黑高

      1. $sizeof(x)>=2^{bh(x)}-1$

        $bh(x)=0$，空树，$sizeof(x)=0$，显然成立

        假设$bh(x)=0,1,…,k$都成立，$bh(x)=k+1$时，$sizeof(x->child)>=2^k-1$，所以$sizeof(x)=2*sizeof(x->child)+1>=2^{k+1}-1$

        成立

      2. $bh(Tree)>=h(Tree)/2$

        红黑树不存在父子关系的红节点，故结论显然成立

      由1,2知结论成立

    insert

    • 插入的节点染成红色，若插入后父亲是红色就要调整
      • 叔红：叔、父与祖父换色
      • 叔黑：
        1. zigzag型：旋转父亲变成zigzig
        2. zigzig型：父亲和祖父换色，旋转祖父（父为轴）

    delete

    • 非叶节点和红叶节点：按照BST删除方法进行删除，但原节点颜色不变（此时不会影响RBT的性质）

    • 黑叶节点（在该路径下要增加一个黑节点）

      • 兄红（case1）：父兄换色父单旋（兄为轴）（未处理完）

      • 兄黑：

        • 双侄黑（case2）：兄变红
          • 父红（case2.1）：父变黑（完成）
          • 父黑（case2.2）：目标节点上移
        • zigzag位的侄子为红，zigzig位的侄子为黑（case3）：红侄子与兄换色，兄单旋（兄为右儿子就右单旋，左儿子就左单旋）
        • zigzig位的侄子为红（case4）：父兄换色，zigzig位侄子置黑，父单旋（兄为轴）（完成）

    RB tree vs AVL tree

    • Number of rotations:

      | | Insert | Delete |
      | —— | ——— | ———- |
      | RB | <=2 | <=3 |
      | AVL | <=2 | O(logN) |

    左式堆

    • 对于堆中任一节点X，$Npl(X->leftchild)>=Npl(X->rightchild)$
    • $Npl(X)$为X到无双儿节点的最短路径长
    • merge的时间界为$O(logN)$

    定理1

    • 右路径上有$r$个节点的左式堆至少有$2^r-1$个节点

      Proof by induction

      $r=1$时，显然堆至少有一个节点，成立

      假设$r=1,2,…,k$时均成立，$r=k+1$时，

      先考虑根的右子树，此时右子树的右路径上有$r$个节点，那么右子树至少有$2^r-1$个节点

      再考虑左子树，此时左子树的右路径至少有$r$个节点（否则根不是一个左式堆），那么左子树也至少有$2^r-1$个节点

      故总节点数至少为$2^r-1+2^r-1+1=2^{r+1}-1$，由归纳法知成立

    merge($O(logN)$)

    • 递归

      1. H1或H2空时，返回另一个

      2. 不妨设H1->val < H2->val

        • H1->left == NULL，H1->left = H2

        • H1->left != NULL，合并H1->right和H2

      3. 不符合左式堆结构时交换新堆的左右子树

      4. 更新新堆的Npl值

    • 非递归
      1. 将H1和H2右路径上的键值按堆序性排序合并
      2. 遇到不符合左式堆结构时交换左右子树

    deletemin

    1. 删除根节点
    2. merge左右子树

    斜堆

    merge(amortized bound: $O(logN)$)

    • 不维护Npl值，每次merge都交换左右子树（除了右路径上最大键值的子树）
    • 除了上面一点，操作与左式堆一样
    • 将1到$2^k-1$按顺序插入到一个斜堆或左式堆中会得到一棵满二叉树

    二项队列

    • 由堆序性的树构成的森林
    • 二项树用$B_k$表示，k代表二项树的高度，其中$B_k$有$2^k$个节点
    • 二项树$B_k$在第d层的节点数为$C_k^d$
    • 一个二项队列可用一个二进制串表示，merge操作实质上是在做二进制加法

    operations

    findmin

    • 最小节点必在某一棵二项树的根节点
    • 故$T(N)=O(logN)$
    • 但可以随时记录并更新最小节点，维护一个指向最小节点的指针，此时$T(N)=O(1)$

    merge

    • merge操作的实质是二进制加法
    • 故$T(N)=O(logN)$
    • 上述操作的前提是二项队列里的二项树按高度排序

    insert

    • insert操作是特殊的merge操作，将N个元素插入到一个空堆的操作复杂度为$O(N)$，每次insert的$T(N)=O(1)$

    • proof

      • 聚合分析

        • 将一系列插入操作看成二进制数从零自增的过程

        • N次操作需要加N次——$O(N)$

        • 接着考虑进位操作

          N次操作共进位$N(1/4+21/8+31/16+…N*1/2^{N+1})$

          括号里无穷级数收敛，求和后为一个常数，故进位操作的复杂度仍为$O(N)$

      • 势能分析

        • $\Phi_i$代表i次insert后堆中树的个数
        • $c_i+\Phi_i-\Phi_{i-1} = 1+1=2$，前面的1代表节点加进去的操作，后面的1代表若进行了k次进位，树的个数减少k-1
        • 故时间复杂度为$O(N)$

    Deletemin

    • 步骤
      • findmin——$O(logN)$
      • 假设min节点在$B_k$，将$B_k$从堆中移除——$O(1)$
      • 将$B_k$的根移除，得到其子树构成的另一个二项队列——$O(log(N))$
      • 将原队列与刚才得到的二项队列merge——$O(logN)$

    implementation

    • 堆的主要结构是一个指针数组，其中第k项指向二项树$B_k$
    • 二项树的表示采用左儿子右兄弟，子树从左到右由大到小，这样使combine trees在常数时间内完成

    Divide and conquer

    Master method

    [Master Theorem]

    Let $a\geq1$ and $b > 1$ be constants, let $f(N)$ be a function, and let $T(N)$ be defined on the nonnegative integers by the recurrence $T(N)=aT(\frac{N}{b})+f(N)$. Then:

    1. If $f(N)=O(N^{\log_ba-\varepsilon})$ for some constant $\varepsilon > 0$, then $T(N)=\Theta(N^{\log_ba})$
    2. If $f(N) = \Theta(N^{\log_ba})$, then $T(N) = \Theta(N^{\log_ba}\log{N})$
    3. If $f(N)=\Omega(N^{\log_ba+\varepsilon})$ for some constant $\varepsilon > 0$, and if $af(\frac{N}{b}) < cf(N)$ for some constant $c<1$ and all sufficiently large $N$, then $T(N) = \Theta(f(N))$

    However, this cannot cover all the cases.

    [Master Theorem] another form

    The recurrence $T(N) = aT(\frac{N}{b})+f(N)$ can be solve as follow:

    1. If $af(\frac{N}{b})=\kappa f(N)$ for some constant $\kappa<1$, then $T(N)=\Theta(f(N))$
    2. If $af(\frac{N}{b})=\Kappa f(N)$ for some constant $K>1$, then $T(N)=\Theta(N^{\log_ba})$
    3. If $af(\frac{N}{b})=f(N)$, then $T(N)=\Theta(f(N)\log_bN)$
    [Theorem]

    The solution to the equation $T(N)=aT(\frac{N}{b})+\Theta(N^k\log^pN)$

    where $a\geq1$,$b\geq1$,and $p\geq0$ is

    1. $T(N)=O(N^{\log_ba})$ if $a>b^k$
    2. $T(N)=O(N^k\log^{p+1}N)$ if $a=b^k$
    3. $T(N)=O(N^k\log^pN)$ if $a<b^k$
    example

    $T(n) = 2T(\sqrt n) + logn$

    此时无法直接用master theorem，需要转化

    令$n = 2^m$

    则化为$T(2^m) = 2T(2^{m/2})+m$

    令$G(m) = T(2^m)$

    则$G(m) = 2G(m/2)+m$

    由master theorem 知$T(n) = G(m) = O(mlogm) = O(lognloglogn)$

    NP-Completeness

    Turing machine

    • Deterministic Turing Machine: executes one instruction at each point in time. The depending on the instruction, it goes to the next unique instruction
    • Nondeterministic Turing Machine: free to choose its next step from a finite set. And if one of these steps leads to a solution, it will always choose the correct one

    NP

    • Solve by a ==N==ondeterministic Turing Machine in ==p==olynomial time
    • proof any solution is true in polynomial time(equal to the above one)

    NP-Complete

    • The hardest in NP
    • Any problem in NP can be polynomially reduced to it(we can convert any NP problem to a NPC problem in polynomial time)
    • NP hard >= NPC !!!

    Approximation

    target

    • find near-optimal solutions in a polynomial time

    approximation ratio

    • 任意规模n的输入，$max(C/C^,C^/C)<=\rho(n)$，其中$C^*$为最优解，$C$为近似解。

    approximation scheme

    • 也属于一种近似算法

    • 输入多了一个值$\epsilon>0$，使得对于任意固定的$\epsilon$，该模式是一个$(1+\epsilon)$- approximation

    • 如果对于任意的$\epsilon$，该approximation scheme在多项式时间内完成，则称之为PTAS(polynomial-time approximation scheme)。

      但常常是$O(n^{2/\epsilon}),O((1/\epsilon)^2n^3)$这样的复杂度，当算法结果越接近精确值，时间复杂度也将迅速增大，$O((1/\epsilon)^2n^3)$被称为FPTAS，因为对于n和$\epsilon$都是polynomial的

    Bin packing

    next fit

    • 对于每个物品，上一个箱子能放下就放下，放不下就开个新箱子
    • $O(n)$
    • 假设最少需要M个箱子，则next fit最多需要2M-1个箱子，而且这个上界可以取到

    first fit

    • 对于每个物品，从头遍历每个箱子，遇到能放进去的就放进去，要是都放不进去就开一个新箱子
    • $O(nlogn)$
    • 假设最少需要M个箱子，first fit最多要17M/10个箱子，总能构造出一个序列使其用掉17(M-1)/10个箱子

    best fit

    • 寻找能放进而且剩余空隙最小的箱子
    • $O(nlogn)$
    • 和first fit相同，上界仍然是1.7M

    on-line algorithm

    • 对于在线装箱问题，总存在一个序列使得算法用掉的箱子至少是最优解的5/3倍

    off-line algorithm

    • 先将物品大小降序排列，在运用first fit或best fit
    • M是最优解，对于off-line的first fit，最多用掉(11M+6)/9个箱子

    knapsack problem

    fractional version

    • 给一个容量为M的箱子和N件物品，第i件物品重量为wi，利润为pi，wi的xi被打包，即可获利pixi，找到获利最大的打包方案，即$\sum_{i=1}^Nw_ix_i<=M$的情况下，使$\sum_{i=1}^Np_ix_i$最大化

    • greedy：依pi/wi的值对物品进行降序排列，然后依次选取

    0-1 version

    • 每个物品要么取要么不取
    • greedy：对pi或pi/wi进行贪心——2-approximation
    • dp：经典dp问题

    K-center problem

    问题

    • 输入n个sites：$s_1,…,s_n$，选择K个centers，此时每个site都有一个离它最近的center，记$s_i$对应的这个距离为$dist(s_i,C)$，目标是使得$r(C)=\max_i dist(s_i,C)$最小

    greedy

    • 假设最优解为$r(C^)$，那么任取一个site当作center时，该半径不会超过$2r(C^)$

    • ```pseudocode
      Centers Greedy-2r ( Sites S[ ], int n, int K, double r )
      { Sites S’[ ] = S[ ]; / S’ is the set of the remaining sites /

      Centers  C[ ] = empty;
while ( S’[ ] != empty ) {
    Select any s from S’ and add it to C;
    Delete all s’ from S’ that are at dist(s’, s) <= 2r;
} /* end-while */
if ( |C| <= K ) return C;
else ERROR(No set of K centers with covering radius at most r);

      }

    • 上述算法仍然是2-approximation

    • Unless P=NP，不存在K- center的$\rho$- approximation算法使得$\rho<2$

    Local search

    neighbor relaiton

    • $S\sim S’$: $S’$ is a neiboring solution of S($S’$可由S的微小改动得到)
    • $N(S)$: $S’$组成的集合

    vertex cover

    • 寻找最小点集实现顶点覆盖

    • feasible solution set $\mathcal F\mathcal S$: all the vertex covers($V \in \mathcal F\mathcal S$)

    • cost(S) = |S|
    • $S\sim S’$: $S’$由S增加或删除一个点得到
    • search：从S=V开始，不断删除点，直到不是顶点覆盖（很多时候无法到达全局最优）

    Hopfiled Neural Networks

    定义

    • 图G具有非0整数权重w的边
    • 顶点具有两种状态：+1和-1

    • 边权为负：两顶点状态相同；边权为正：两顶点状态相反

    • 边权的绝对值代表我们想要两顶点状态相同或相反的强烈程度

    • 输出：a configuration S: 给每个顶点assign一个state满足上述条件

    问题

    • 上述configuration不一定能找到，所以退而求其次，找到一个sufficiently good的解

    • edge e(u, v) is good：$w_es_us_v<0$

      上述条件等价于：$w_e<0$当且仅当$s_u=s_v$

    • node u is satisfied: 在包含该顶点的所有边中，好边总权值大于等于坏边总权值

      公式表达：$\sum_{v:e(u,v)\in E}w_es_us_v<=0$

    • a configuration is stable: 所有顶点satisfied

    state-flipping algorithm

    • 顶点可能被flip多次，但此算法一定会终止
    • Claim: state-flipping算法最多循环$W=\sum_e|w_e|$次

    maximum cut

    定义

    • 对于边权为正整数的无向图G，将顶点分成两部分A, B，使得$w(A, B)=\sum_{u\in A,v\in B}w_{uv}$最大

    related to local search

    • Problem: maximize $w(A,B)$
    • Feasible solution set: any partition (A, B)
    • $S\sim S’$: $S’$可由S将一个顶点由一个集合移到另一个集合得到

    special Hopfield Neural Network

    • 上述问题是特殊的Hopfield，所有边权大于0
    • 但用state-flipping解时，不一定是全局最优

    how good is the local optimum

    • Claim: $(A,B)$为局部最优，$(A^,B^)$为全局最优，则$w(A,B)>=w(A^,B^)/2$

      Proof: 任取$u \in A$，因为$(A,B)$为局部最优，故有$\sum_{v \in A}w_{uv}<=\sum_{v\in B}w_{uv}$

      两边对所有$u\in A$求和：

      $\sum_{u\in A}\sum_{v \in A}w_{uv}<=w(A,B)$

      同理：

      $\sum_{u\in B}\sum_{v \in B}w_{uv}<=w(A,B)$

      所以$w(A^,B^)<=\sum w_e=w(A,B)+1/2\sum_{u\in A}\sum_{v \in A}w_{uv}+1/2\sum_{u\in B}\sum_{v \in B}w_{uv}<=w(A,B)+1/2w(A,B)+1/2w(A,B)=2w(A,B)$

    • Unless P=NP, no 17/16 approximation algorithm for MAX-CUT

    improvement

    • 上述算法可能无法在多项式时间内完成，可以考虑提早跳出循环：当任何翻转后都无法提升“足够多”时，就跳出循环
    • Big-improvement-flip：选择node翻转时保证翻转后$w(A,B)$至少提升$\frac{2\epsilon}{|V|}w(A,B)$
    • 上述算法结束循环时，能够保证$(2+\epsilon)w(A,B)>=w(A^,B^)$
    • 上述算法结束循环时，最多执行$O(\frac{n}{\epsilon} logW)$次翻转
