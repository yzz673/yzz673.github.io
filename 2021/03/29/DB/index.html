    relational model

    key

    • 超码：一个或多个属性的集合，可以唯一标识一个元组
    • 候选码：最小超码
    • 主码：被设计者选中的候选码
    • 外码：r1的属性中包含r2的主码，称为r1参照r2的外码。外码需遵循参照完整性约束，即对于参照属性，r1中出现的的取值在r2的对应属性中必然要出现。

    SQL

    • smallint: 小整数类型
    • numeric(p, d): 共p 位数字（一位符号位），d位数字在小数点右边
    • float(n): 精度至少位n
    • real: 浮点数
    • double precision: 双精度浮点数

    字符串操作

    • “s%”匹配任何以字符串”s”打头等字符串
    • “% s%”匹配任何包含”s”的字符串
    • “_ _ _”匹配只含三个字符的字符串

    • “_ _ _%”匹配至少含三个字符的字符串

    • like “ab\%cd%” escape ‘\ ‘匹配所有以”ab%cd”开头的字符串

    并、交、差运算

    • union，intersect，except运算自动去重
    • 若要不去重，则在前面加上all
    • 对于差运算，若不去重，则结果中某属性的个数位被减量中出现的次数减去减量中出现的次数

    分组聚集和having子句

    • 若有些属性在select子句中出现但没有在group by子句中出现，则该属性必须出现在聚集函数内部，不然会报错
    • having子句应用到每个分组上，不满足having子句谓词的分组会被抛弃
