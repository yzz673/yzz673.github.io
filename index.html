traditional feature-based methods

Node-level

Eigenvector centrality

  • a node v is important if surrounded by important neighboring nodes u $\in$ N(v)

    ——我的邻居越重要，我就越重要

  • $c_v = \frac{1}{\lambda}\sum_{u\in N(v)}c_u \leftrightarrow \lambda c=Ac$

    • A是adjacency matrix
    • c是central vector

  • $\lambda_{max}$恒为正值且唯一

  • $c_{max}$is used for centrality

betweenness centrality

  • a node v is important if it lies on many shortest paths between other nodes
  • $c_v = \sum_{s\neq v\neq t}\frac{n_v}{n}$, $n_v$ is the shortest paths between s and t contains v, n is the shortest paths between s and t

closeness centrality

  • a node v is important if it has small shortest path lengths to all other nodes
  • $c_v = 1/\sum_{u\neq v}l_{uv}$, $l_{uv}$is the length of the shortest path between u and v

clustering coefficient

  • measures how connected v’s neighboring nodes are
  • $e_v = E_v/C_{k_v}^2 \in [0,1]$, $E_v$ is the edges among neighboring nodes, $k_v$ is the degree of v
  • clustering coefficient counts the #(triangles) in the ego-network

graphlets

  • rooted connected non-isomorphic subgraphs
  • graphic degree vector(GDV): graphite-base features for nodes, a count vector of graphlets rooted at a given node

Link-level

distance-based features

  • Shortest-path distance between 2 nodes——does not capture the degree of neighborhood overlap

local neighborhood overlap

  • Capture neighboring nodes shared between v1 and v2
    • Common neighbors: $|N(v_1) \cap N(v_2)|$
    • Jaccard’s coefficient: $|N(v_1) \cap N(v_2)|/|N(v_1) \cup N(v_2)|$
    • Adamic-Adar index: $\sum_{u \in N(v_1) \cap N(v_2)}1/logk_v$

Global neighborhood overlap

  • Katz index: count the number of paths of all lengths between a given pair of nodes
  • 用邻接矩阵的幂计算路径数，即$A^k_{uv} = P^{(k)}_{uv}$
  • Katz index between v1 and v2 is calculated as $S_{v_1v_2}=\sum_{l=1}^{\infty}\beta^lA^l_{v_1v_2}, \beta \in (0,1)$, $\beta$是一个权重因子，距离越远权重越低
  • Katz index matrix: $S = \sum_{i=1}^{\infty}\beta^iA^i=(I-\beta A)^{-1}-I$

Graph-level

kernel methods

  • Kernel $K(G,G’) \in R$ measures similarity between data points(2 graphs)
  • Kernel matrix $K = (K(G,G’))_{G,G’}$必须半正定

  • There exists a feature representation $\phi(.)$ such that $K(G,G’)=\phi(G)^T \phi(G’)$

    graphlet kernel

  • key ides: count the number of different graphlet in a graph

  • graphlet here is different from node-level features:

    • node in graphlet do not need to be connected
    • the graphlet are not rooted

  • Given graph G

    • the graphlet list $\mathcal{G_k} = (g_1,…g_{n_k})$
    • the graphlet count vector $f_G \in R^{n_k}, (f_G)_i = num(g_i \subseteq G)$
  • Graphlet kernel: $K(G,G’) =h_G^Th_{G’}, h_G = f_G/sum(f_G)$

weisfeiler-lehman(WL) kernel

  • Color refinement
    • Given a graph G with a set of nodes V
      • Assign a color $c^{(0)}(v)$ to each node v
      • Iteratively refine node colors by $c^{(k+1)}(v) = HASH(\{c^{(k)}(v),\{c^{(k)}(u)\}_{u \in N(v)}\})$, HASH maps different inputs to different colors
      • After K steps of color refinement, $c^{(K)}(v)$ summarizes the structure of K-hop neighborhood
  • After refinement, WL kernel counts number of nodes with a given color——形成一个向量$\phi(G)$

Node embedding

step

  • Encoder maps from nodes to embeddings

  • Define a node similarity function(a measure of similarity in the original network)

  • Decoder DEC maps from embeddingds to the similarity score

  • Optimize the parameters of the decoder so that:

    $similarity(u, v) \approx z_v^Tz_u$

encoder

  • Maps each node to a low-dimensional vector

    $ENC(v) = z_v$

  • Simplest approach: an embedding-lookup

random walk

  • notation

    • $z_u$: the embedding of node u
    • $P(v|z_u)$: the (predicted) probability of visiting node v on random walks starting from node u
    • $\sigma(z)_i=e^{z_i}/\sum_{j=1}^Ke^{z_j}$: Softmax function, turns vector of K values(model predictions) into K probabilities that sum to 1
    • $S(x)=1/(1+e^{-x})$: sigmoid function, S-shaped function that turns real values into range of $(0,1)$

  • random walk: given a graph and a starting point, select a neighbor randomly then move to the neighbor. The sequence of point visited this way is a random walk on the graph

  • random walk embeddings

    • $z_u^Tz_v \approx $ probability that u and v co-occur on a random walk over the graph
    • Idea: if random walk starting from node u visits v with high probability, u and v are similar

  • Unsupervised feature learning

    • Intuition: find embeddings of nodes in d-dimensional space that preserves similarity
    • Idea: learn node embedding such that nearby nodes are close together in the network

  • feature learning as optimization:

    • Given G(V, E)

    • Goal: learn a mapping $f:u \rightarrow R^d:f(u)=z_u$

    • 似然函数(Log-likelihood objective):

      $max_f\sum_{u \in V}logP(N_R(u)|z_u)$

      $N_R(u)$ is the neighborhood of node u by strategy R

  • random walk optimization

    • run short fixed-length random walks starting from each node u in the graph using some random walk strategy R
    • for each node u collect $N_R(u)$, the multiset of nodes visited on random walks
    • optimize embeddings according to: given node u, predict its neighbor $N_R(u)$
    • 似然函数：$L=\sum_{u \in V}\sum_{v \in N_R(u)}-log(P(v|z_u))$
    • 定义$P(v|z_u)=exp(z_u^Tz_v)/\sum_{n \in V}exp(z_u^Tz_n)$
    • 按上述定义，时间复杂度过高，需要优化$P(v|z_u)$
    • 优化方法：negative sampling
      • $log(exp(z_u^Tz_v)/\sum_{n \in V}exp(z_u^Tz_n)) \approx log(\sigma(z_u^Tz_v))-\sum_{i=1}^klog(\sigma(z_u^Tz_{n_i})$
      • instead of normalize all nodes, just normalize against k random negative samples
      • Choose negative samples
        • Sample k negative node each with probability, proportional to its degree(度数更高更有可能被选作negative sample)
        • Higher k gives robust estimates
        • Higher k corresponds to higher bias on negative events
        • In practice k = 5-20

  • node2vec

    • Goal: embed node with similar network neughborhoods close in the feature space
    • Develop biased 2nd order random walk R to generate network neighborhood $N_R(u)$ of node u
    • use DFS or BFS
    • Interpolating BFS an DFS
      • 2 parameters
        • return parameter p
        • in-out parameter q

  • embedding entire graph

    • Anonymous walk

      • 对定点进行索引编号，索引为第一次遇到该点的时间

      • example：

        A->B->C->B->C对应1->2->3->2->3

        同样的，C->D->B->D->B也对应1->2->3->2->3

      • Anonymous walks 的个数与点个数呈指数关系

    • sampling anonymous walks

      • generate independently a set of m random walks
      • 要想使误差大于$\epsilon$的概率小于$\delta$，$m=\lceil \frac{2}{\epsilon^2}(log(2^\eta-2)-log\epsilon)\rceil$($\eta$ is the total number of anonymous walks of length $l$)

PageRank

PageRank: The “flow” Model

  • Idea: links as votes

  • A vote from an important page is worth more

    • Each link’s vote is proportional to the importance of its source page
    • If page i with importance ri has di out-links, each links gets ri/di votes
    • Page j’s own importance rj is the sum of the votes on its in-links

  • node j 的 rank rj = $\sum_{i\rightarrow j}r_i/d_i$，其中di是node i的out-degree

Matrix Formulation

  • Stochastic adjacency matrix M

    • Page j have dj out-links
    • if j -> i, then Mij = 1/dj (M is a column stochastic matrix, column sum to 1)

  • Rank vector r: An entry per page

    • ri is the importance score of page i
    • $\sum_ir_i = 1$

  • 上面的flow equation可以写成r = M*r

connection to random walk

  • 假设有一个random web surfer

    • 在t时刻，surfer停留在page i
    • 在t+1时刻，surfer等可能走到其中一个out-link
    • 重复上述过程

  • 令p(t)为一个概率向量，其中第i项是是这个surfer在时间t停留在页面i的概率(表示概率分布)

    p(t+1) = M*p(t)

    当不断迭代到p(t) = p(t+1)时，p(t)称为stationary distribution of a random walk

    由上面知r = M*r，所以r就是stationary distribution

  • 从特征值的角度来看，r是M的特征向量(特征值为1)

    任取初始向量u，不断左乘M，最后收敛到r

The Best Peak Shape

introduction

In many research areas, one important target of analyzing data is to find the best “peak shape” out of a huge amount of raw data full of noises. A “peak shape” of length $L$ is an ordered sequence of numbers $D_1,…,D_L$ satisfying that there exists an index $i(1D_{i+1}>…>D_L$.

Now given $N$ input numbers ordered by their indices, you may remove some of them to keep the rest of the numbers in a peak shape. The best peak shape is the longest sub-sequence that forms a peak shape. If there is a tie, then the most symmetric (meaning that the difference of the lengths of the increasing and the decreasing sub-sequences is minimized) one will be chosen.

Input Specification

Each input file contains one test case. For each case, the first line gives an integer $N(3<=N<=10^4)$. Then $N$ integers are given in the next line, separated by spaces. All the integers are in [−10000,10000].

Output Specification

For each case, print in a line the length of the best peak shape, the index (starts from 1) and the value of the peak number. If the solution does not exist, simply print “No peak shape” in a line. The judge’s input guarantees the uniqueness of the output.

solution

idea

  • 一道很简单的dp问题，本质是最长单调子序列
  • 正向求单增子列，逆向求单减子列
  • 再遍历一遍找出结果

safe fruit

问题描述

There are a lot of tips telling us that some fruits must not be eaten with some other fruits, or we might get ourselves in serious trouble. For example, bananas can not be eaten with cantaloupe (哈密瓜), otherwise it will lead to kidney deficiency (肾虚).

Now you are given a long list of such tips, and a big basket of fruits. You are supposed to pick up those fruits so that it is safe to eat any of them.

Input Specification:

Each input file contains one test case. For each case, the first line gives two positive integers: N, the number of tips, and M, the number of fruits in the basket. both numbers are no more than 100.

Then two blocks follow. The first block contains N pairs of fruits which must not be eaten together, each pair occupies a line and there is no duplicated tips; and the second one contains M fruits together with their prices, again each pair in a line. To make it simple, each fruit is represented by a 3-digit ID number. A price is a positive integer which is no more than 1000. All the numbers in a line are separated by spaces.

Output Specification:

For each case, first print in a line the maximum number of safe fruits. Then in the next line list all the safe fruits, in increasing order of their ID’s. The ID’s must be separated by exactly one space, and there must be no extra space at the end of the line. Finally in the third line print the total price of the above fruits. Since there may be many different solutions, you are supposed to output the one with a maximum number of safe fruits. In case there is a tie, output the one with the lowest total price. It is guaranteed that such a solution is unique.

题意解读

  • 水果对应ID号不一定连续，需要对ID号与水果对应索引进行一个双射，减少空间浪费，具体方法是维护两个向量id2idx和idx2id，前者是ID号到索引的映射，后者是索引到ID号到映射
  • tips中出现的水果在basket里有可能不会出现，故需要注意
  • 需要用dp进行剪枝，不然超时

解题思路

最大独立集问题

  • 将水果看成点，如果两个水果不能一起吃则它们之间有一条无向边

  • 此题是一道裸最大独立集问题，即找出一个最大的集合，使得集合中的任意两个点都没有边

基本算法

  • 回溯算法，对所有点进行搜索
  • 对于某个点u，初始点集合为所有不与u相邻的点，记为P，易知包含u的最大独立集中的点均取自P
  • 搜到某个点v 时，从P中遍历不与v相邻接的点，对这些点继续搜索

剪枝

  • 对每个点搜索时采用倒序，即从索引大的向小的搜索
  • 维护一个数组dp，dp[i]为索引i到m-1的点集组成的最大独立集的点数
  • 根据以上内容可在回溯过程中增加两个剪枝条件，都是在确定再搜索下去不能产生更大的独立集的情况下return，具体见代码部分

functional dependency

  • 给定一个实例r(R)，a, b为属性集。a->b，即a决定b

    对于任意两个元组t1, t2，若t1[a] = t2[a]，则一定有t1[b] = t2[b]

  • 若k->R，则k是superkey

Armstrong’s axioms

  • 3 axioms:

    • Reflexive: if b $\subseteq$ a, then a->b
    • Augmentation: if a->b, then ac->bc
    • Transitivity: if a->b and b->c, then a->c

  • Armstrong’s axioms can be used repeatedly to compute $F^+$

additional rules

  • 3 rules:

    • Union: if a->b and a->c, then a->bc
    • Decomposition: if a->bc, then a->b and a->c
    • Pseudotransitivity: if a->b and bc->d, then ac->d

Lossless Decomposition

  • 将r(R)分解为r1(R1)和r2(R2)时未失去任何信息

  • 将R分解为R1，R2时，若

    • R1 $\cap$ R2 -> R1
    • R1 $\cap$ R2 -> R2

    中至少有一个在$F^+$中，则R1, R2是无损分解，即R1 $\cap$ R2是R1或R2的超码，则为无损分解

Dependency Preservation

  • 将R分解为R1,…,Rn; F对Ri对restriction为Fi, 其中Fi $\subseteq$ F 且Fi只含Ri中的属性
  • 令$F^{‘}$为F1到Fn的并，若$F^{‘+}=F^+$，则分解为保持依赖的

Boyce-Codd Normal Form(BCNF)

  • 关系模式R的依赖集F满足BCNF的条件是：对所有functional dependency a->b in $F^+$，至少满足以下两个条件之一：

    • b $\subseteq$ a
    • a是R的一个superkey

  • 对于不满足BCNF的schema R，想要decompose R时，由于至少存在函数依赖a->b不满足BCNF的两个条件，将R分为两个部分

    • a $\cup$ b
    • R-(b-a)

Third Normal Form

  • 关系模式R的依赖集F满足第三范式的条件是：对所有functional dependency a->b in $F^+$，至少满足以下三个条件之一：

    • b $\subseteq$ a
    • a是R的一个superkey
    • 对于b-a中的每一个attribute A，A都在R的某个候选码中

multivalued dependency

  • 若a->b，则a->->b
  • 若a->->b，则a->->R-a-b
  • 若b $\subseteq$ a或b $\cup$ a = R，则a->->b是平凡的

第四范式

    • a->->b是平凡的
    • a是superkey

储存和文件结构

储存介质

  • cache：高速缓冲储存器，位于CPU和主存储器DRAM（Dynamic Random Access Memory）之间，规模较小，但速度很快，通常由SRAM（Static Random Access Memory静态存储器）组成
  • main memory：主储存器，存放可处理的数据，通用机器指令在主储存器上执行
  • flash memory：快闪储存器，具有非易失性，目前有两种类型：NAND快闪和NOR快闪
  • magnetic-disc storage：磁盘储存器，用于长期联机数据存储
  • optical storage：光学存储器
  • tape storage：磁带存储器

storage hierarchy

  • Primary storage: 基本存储(cache, main memory)，易失性
  • Secondary storage: 辅助存储(flash memory, magnetic disks)，又叫online storage(联机存储)，非易失性
  • Tertiary storage: 三级存储(magnetic tape, optical storage)，又叫off-line storage(脱机存储)，非易失性，速度慢

磁盘

  • 盘片表面从逻辑上划分为磁道(track)，磁道又划分为扇区(sector)。扇区是磁盘读写的最小单位
  • 读写头(read-write head)：通过反转磁性物质磁化的方向将将信息磁化储存到扇区中
  • 所有盘片的第i条磁道合在一起称为第i个柱面

disc controller

  • interfaces between the computer system and the disk drive hardware
  • accepts high-level commands to read or write a sector
  • 为其所写的扇区附加校验和(checksum)，校验和是从写到扇区中的数据计算得到的。读取一个扇区的数据时，disc controller用读到的数据再次计算一次校验和，与储存的校验和相比较，若不一致，则会重读几次，若一直错误，则发出一个读取失败信号
  • remapping of bad sector：磁盘初始格式化或写一个扇区时，若disc controller检测到一个损坏的扇区则会把这个扇区在逻辑上映射到另一个物理位置，写操作在该位置上执行

Disk Subsystem

  • 将磁盘连接到计算机操作系统的接口
    • ATA (AT adaptor) range of standards
    • SATA (Serial ATA)
    • SCSI (Small Computer System Interconnect) range of standards
    • SAS (Serial Attached SCSI)
    • Several variants of each standard (different speeds and capabilities)

磁盘性能度量

  • access time：访问时间，从发出请求到数据开始传输到时间
  • 访问数据时，磁盘臂必须先定位到正确的磁道，该时间被称为寻道时间(seek time)，平均寻道时间是寻道时间的平均值，依赖于目的磁道与磁盘臂的距离
  • 定位到正确的磁道后，要访问的扇区出现在读写头下的时间称为旋转等待时间(rotational latency time)，平均旋转等待时间=磁盘旋转一周的时间/2
  • 数据传输率(data-transfer rate)，从磁盘获得数据或者向磁盘存储数据的速率
  • 平均故障时间(mean time to failure, MTTF)，这是磁盘可靠性的度量

磁盘块访问的优化

  • buffering：缓冲，从磁盘读取的块暂时存储在缓冲区中
  • read-ahead：预读，当一个磁盘块被访问时，相同磁道的连续块也被读入内存缓冲区中（对于随机块访问不是很有用）
  • scheduling：调度，访问不同柱面上的块时，按照使磁盘臂移动最短距离的顺序发出请求。Disk-arm-scheduling algorithms(磁盘臂调度算法) order pending accesses to tracks so that disk arm movement is minimized
  • file organization：文件组织，按照与预期数据访问方式最接近的方式来组织磁盘上的块
  • nonvolatile write buffer：非易失性写缓冲区，加速写磁盘的操作
  • log disc：日志磁盘，专门用于写顺序日志的磁盘，消除了寻道时间

RAID

  • redundancy：冗余，存储正常情况下不需要的额外信息，但这些信息可在磁盘故障时重建丢失的信息。最简单也是最昂贵的一种方法是镜像技术，即复制每张磁盘的信息。
  • 并行，在多张磁盘上进行数据拆分(striping data)来提高传输速率
    • 比特级拆分(bit-level striping)，将一个byte拆分成8个bit到不同的磁盘上（适用于磁盘总数为8的倍数或8的因子的情形），读到效率为原来的8倍但写和访问的效率降低
    • 块级拆分(block-level striping)，将块拆分到多张磁盘，假设将块从0开始编号，一共n张磁盘，则第i个块存储在第(i mod n)+1张磁盘上

系数

这种表示形式中$1,x,x^2,…,x^n$作为一组基

差值中假设有$P_n(x_i) = y_i(0<=i<=n)$，若用上面的一组基表示$P_n$，则需要解方程组

此时要确定Lagrange系数$L_{n,i}$

余项

切比雪夫多项式

问题背景

  • 给定函数$f(x)$，寻找n次多项式$P_n(x)$使得$||P-f||_{\infty}$最小，是一个minimax问题

性质

  • $f \in C[a, b]$且$f$不是n次多项式，则存在一个唯一的n次多项式$P_n(x)$使得$||P-f||_{\infty}$最小
  • Chebyshev Theorem：假设$P_n(x)$使得$||P-f||_{\infty}$最小，等价于函数$P-f$至少有n+2个最值点且最大值和最小值交替出现（且绝对值相等），这n+2个最值点组成的序列叫Chebyshev alternating sequence

问题等价转化

  • 由chebyshev theorem，$P_n(x)$是对$f$的n次差值多项式，所以$P$由n+1个差值点决定，此时转化为寻找合适的差值点问题，此时余项$|R_n(x)| = |\frac{f^{(n+1)}(\xi)}{(n+1)!}\prod_{i=0}^n(x-x_i)|$，$\xi$不可控并且大多数情况下f在n+1次求导后会十分平缓，近似为常数，故优化后面的部分
  • 令$w_n(x) = \prod_{i=0}^n(x-x_i)$，问题转化为求点列$x_i$使得$||w_n(x)||_{\infty}$最小，其中$x \in [-1, 1]$（实际应用时运用线性变换可扩展到实轴上）
  • $w_n(x) = x^n - P_{n-1}(x)$，问题进一步转化为$||x^n - P_{n-1}(x)||_{\infty}$的最小化问题，由chebyshev theorem知，$w_n(x)$在$[-1, 1]$上有交替出现的n+1个最大（小）值点

定理1

  • $N$个内点的红黑树最高$2log(N+1)$

    Proof

    令$sizeof(x)$表示根为x的树的内点个数，$bh(x)$表示根为x的树的黑高

    1. $sizeof(x)>=2^{bh(x)}-1$

      $bh(x)=0$，空树，$sizeof(x)=0$，显然成立

      假设$bh(x)=0,1,…,k$都成立，$bh(x)=k+1$时，$sizeof(x->child)>=2^k-1$，所以$sizeof(x)=2*sizeof(x->child)+1>=2^{k+1}-1$

      成立

    2. $bh(Tree)>=h(Tree)/2$

      红黑树不存在父子关系的红节点，故结论显然成立

    由1,2知结论成立

定理1

  • 右路径上有$r$个节点的左式堆至少有$2^r-1$个节点

    Proof by induction

    $r=1$时，显然堆至少有一个节点，成立

    假设$r=1,2,…,k$时均成立，$r=k+1$时，

    先考虑根的右子树，此时右子树的右路径上有$r$个节点，那么右子树至少有$2^r-1$个节点

    再考虑左子树，此时左子树的右路径至少有$r$个节点（否则根不是一个左式堆），那么左子树也至少有$2^r-1$个节点

    故总节点数至少为$2^r-1+2^r-1+1=2^{r+1}-1$，由归纳法知成立

Master method

[Master Theorem]

Let $a\geq1$ and $b > 1$ be constants, let $f(N)$ be a function, and let $T(N)$ be defined on the nonnegative integers by the recurrence $T(N)=aT(\frac{N}{b})+f(N)$. Then:

  1. If $f(N)=O(N^{\log_ba-\varepsilon})$ for some constant $\varepsilon > 0$, then $T(N)=\Theta(N^{\log_ba})$
  2. If $f(N) = \Theta(N^{\log_ba})$, then $T(N) = \Theta(N^{\log_ba}\log{N})$
  3. If $f(N)=\Omega(N^{\log_ba+\varepsilon})$ for some constant $\varepsilon > 0$, and if $af(\frac{N}{b}) < cf(N)$ for some constant $c<1$ and all sufficiently large $N$, then $T(N) = \Theta(f(N))$

However, this cannot cover all the cases.

[Master Theorem] another form

The recurrence $T(N) = aT(\frac{N}{b})+f(N)$ can be solve as follow:

  1. If $af(\frac{N}{b})=\kappa f(N)$ for some constant $\kappa<1$, then $T(N)=\Theta(f(N))$
  2. If $af(\frac{N}{b})=\Kappa f(N)$ for some constant $K>1$, then $T(N)=\Theta(N^{\log_ba})$
  3. If $af(\frac{N}{b})=f(N)$, then $T(N)=\Theta(f(N)\log_bN)$
[Theorem]

The solution to the equation $T(N)=aT(\frac{N}{b})+\Theta(N^k\log^pN)$

where $a\geq1$,$b\geq1$,and $p\geq0$ is

  1. $T(N)=O(N^{\log_ba})$ if $a>b^k$
  2. $T(N)=O(N^k\log^{p+1}N)$ if $a=b^k$
  3. $T(N)=O(N^k\log^pN)$ if $a<b^k$

    上述条件等价于：$w_e<0$当且仅当$s_u=s_v$

  • node u is satisfied: 在包含该顶点的所有边中，好边总权值大于等于坏边总权值

    公式表达：$\sum_{v:e(u,v)\in E}w_es_us_v<=0$

  • a configuration is stable: 所有顶点satisfied

state-flipping algorithm

  • 顶点可能被flip多次，但此算法一定会终止
  • Claim: state-flipping算法最多循环$W=\sum_e|w_e|$次

maximum cut

定义

  • 对于边权为正整数的无向图G，将顶点分成两部分A, B，使得$w(A, B)=\sum_{u\in A,v\in B}w_{uv}$最大

related to local search

  • Problem: maximize $w(A,B)$
  • Feasible solution set: any partition (A, B)
  • $S\sim S’$: $S’$可由S将一个顶点由一个集合移到另一个集合得到

special Hopfield Neural Network

  • 上述问题是特殊的Hopfield，所有边权大于0
  • 但用state-flipping解时，不一定是全局最优

how good is the local optimum

  • Claim: $(A,B)$为局部最优，$(A^,B^)$为全局最优，则$w(A,B)>=w(A^,B^)/2$

    Proof: 任取$u \in A$，因为$(A,B)$为局部最优，故有$\sum_{v \in A}w_{uv}<=\sum_{v\in B}w_{uv}$

    两边对所有$u\in A$求和：

    $\sum_{u\in A}\sum_{v \in A}w_{uv}<=w(A,B)$

    同理：

    $\sum_{u\in B}\sum_{v \in B}w_{uv}<=w(A,B)$

    所以$w(A^,B^)<=\sum w_e=w(A,B)+1/2\sum_{u\in A}\sum_{v \in A}w_{uv}+1/2\sum_{u\in B}\sum_{v \in B}w_{uv}<=w(A,B)+1/2w(A,B)+1/2w(A,B)=2w(A,B)$

  • Unless P=NP, no 17/16 approximation algorithm for MAX-CUT

improvement

  • 上述算法可能无法在多项式时间内完成，可以考虑提早跳出循环：当任何翻转后都无法提升“足够多”时，就跳出循环
  • Big-improvement-flip：选择node翻转时保证翻转后$w(A,B)$至少提升$\frac{2\epsilon}{|V|}w(A,B)$
  • 上述算法结束循环时，能够保证$(2+\epsilon)w(A,B)>=w(A^,B^)$
  • 上述算法结束循环时，最多执行$O(\frac{n}{\epsilon} logW)$次翻转
